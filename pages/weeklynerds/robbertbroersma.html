Forms + accessibility

Aantekeningen

Een goed formulier volgt een vaste opbouw: intro → vragen → controle → foutoverzicht → bevestiging. Vraag alleen noodzakelijke gegevens. Verplicht/niet verplicht altijd in tekst aangeven, niet alleen met een asterisk. Zichtbare labeltekst moet altijd matchen met het aria-label of title attribuut.

"Beschrijving en foutmelding koppelen via aria-describedby — de volgorde is: label → beschrijving → foutmelding → invoerveld."

Minimale componentgrootte is 44×44px. Gebruik bdi translate="no" op de controlepagina om gebruikersinvoer niet te laten vertalen. Logical CSS properties zijn belangrijk voor RTL-taalondersteuning. Elk element moet focusbaar zijn en fouten moeten automatisch worden voorgelezen door screenreaders.
