Aantekeningen

1 · Marleen — Medior web developer

Marleen werkt als medior web developer bij De Voorhoede. Ze bouwde onder andere een interactieve website voor het DGTL festival, waarbij ze technieken zoals mix-blend-mode en clip-paths toepaste. De Voorhoede doet puur het bouwen — niet het design. Klanten plaatsen zelf content via een headless CMS zoals DatoCMS.

Klanten die technisch niet aangelegd zijn stellen soms onduidelijke vragen, wat kan leiden tot interessante situaties. Technische zaken uitleggen aan niet-technische klanten is een vak op zich. Een baan als web developer is dan ook niet alleen coderen — menselijke en communicatieve skills zijn minstens zo belangrijk.

2 · Peter — Accessibility expert

Peter is senior front-end developer en de accessibility expert binnen De Voorhoede. Hij vertelde over het NL Design System en hoe De Voorhoede meewerkt aan het herbouwen van een website door componenten toe te voegen aan Storybook. Elk nieuw component moet direct werken voor meerdere potentiële klanten tegelijk.

De styling verloopt via design tokens: twee kanten van CSS — de kant waar je de waardes instelt en de kant waar ze van toepassing zijn. Via een JSON-bestand kun je de waardes per huisstijl aanpassen. Voor elke story rekent Peter 20% extra tijd voor toegankelijkheid, omdat dingen altijd complexer blijken dan je in eerste instantie denkt.

"Door jouw code heb je fysieke impact op iemand."

3 · Bas de Greeuw — Offline-first

Bas introduceerde het offline-first design pattern aan de hand van een PWA voor liftinspecties — in een lift heb je geen internet. Het patroon draait om twee kernprincipes: start bij geen verbinding, en sla lokaal op om later te synchroniseren.

Voordelen: altijd toegang (ook in het vliegtuig), snellere interface zonder wachttijd en besparing op data en batterij. Uitdagingen zijn conflictbeheer (wie heeft autoriteit?), slimme caching en data pruning, en duidelijke sync-statusfeedback aan de gebruiker.

De architectuur loopt van het user device via local storage (IndexedDB) door een sync layer naar de cloud server. De drie lagen zijn interceptie via service workers, opslag via IndexedDB, en synchronisatie. Libraries zoals Dexie maken dit beheerbaar. Let wel op merge conflicts bij meerdere apparaten per gebruiker. Kies offline-first wanneer slechte connectiviteit verwacht wordt, kritieke data altijd beschikbaar moet zijn, of bij productiviteits- en field tools.
