The history of browsers

Wat werd er verteld?

Peter vertelde over de geschiedenis van browsers, van de eerste webbrowser tot de moderne browsers. Hij praatte over de ontwikkeling van verschillende browsers en hun invloed op het web. Daarbij ook wat technische punten zoals de verschillen tussen de rendering engines.

Wat neem ik mee?

Voor deze nerd was er een meer technisch onderwerp, de history van browsers. Een onderwerp waar ik niet veel over weet behalve dat Internet Explorer een oude populaire browser is. Het principe wat ik meeneem is het principe van backwards compatibility: als het ooit heeft gewerkt in een browser zal het altijd blijven werken. Je hoeft je dus geen zorgen te maken of bepaalde functies op je websites blijven werken. Daarnaast gebruikte ik wel eens defer en de script-tag onderaan door elkaar om javascript in te laden, maar weet ik nu dat defer de moderne (en betere) oplossing is.

Het was een nogal langdradig verhaal, de history van browsers, omdat hij letterlijk elke browser langs ging, maar Peter is erg goed in het vertellen en bracht het verhaal gepassioneerd over op een leuke manier.

Aantekeningen

"Als iets ooit in een browser heeft gewerkt zal het voor eens en altijd werken."

Hoe werkt een browser?

Een browser bestaat uit meerdere onderdelen: een rendering engine die HTML en CSS parsed en de DOM/AOM opbouwt, een aparte JavaScript engine, een interface en het Accessibility Object Model dat reageert op ARIA. De rendering engine (Blink/Chromium of Gecko) is niet verantwoordelijk voor de interface of de JS engine.

JavaScript bestaat uit twee lagen: de JS core (if-statements, for-loops etc.) en de DOM: de manier waarop JS een HTML-pagina kan aanspreken en wijzigen.

JavaScript laden

  • script src — do it now
  • script defer src — do it later
  • script async src — i don't care when you do it, just not now

Backward compatibility

CSS is altijd render-blocking. Backward compatibility betekent dat wat ooit werkte altijd blijft werken. Denk aan form.object.elements, body bgcolor en frameset (waarvan iframe een afstammeling is). Wees voorzichtig met layout-wijzigingen, gebruik media queries en laat het daarna met rust. Pas op voor "popcorn layout": elementen die verspringen.

Browsergeschiedenis

WWW Browser (1990) → Mosaic (1993) → Netscape (1994) → Internet Explorer (1995) → Opera (1995) → Safari (2003) → Firefox (2004) → Safari iOS (2007) → Chrome (2008) → Ladybird (2024).
