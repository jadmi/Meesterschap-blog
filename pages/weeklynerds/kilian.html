The rule of the least power

Wat werd er verteld?

Kilian Valkhof vertelde over de "rule of the least power" en hoe deze toegepast kan worden. HTML eerst, dan CSS, dan JS. Javascript is krachtig maar zwaar voor de browser, dus kijk naar alternatieven met HTML/CSS. Hierbij gaf hij verschillende voorbeelden van JS toepassingen die nu met HTML/CSS te maken zijn.

Wat neem ik mee?

Als kick-off van de weekly nerds vond ik het wel een interessante spreking, zeker als je kijkt naar het gebruik van progressive enhancement is de rule of the least power een goede om mee te nemen. Choose the least powerful language suitable for a given purpose, dit scheelt ook weer performance. En is iets om wat vaker bij stil te staan.

Ik moet bovendien ook denken aan CSS to the rescue, waarbij je er eigenlijk ook achter komt dat er veel meer mogelijk is met CSS (gecombineerd met HTML) dan je zou denken.

Aantekeningen

Achtergrond

Kernboodschap: gebruik de minst krachtige taal die het werk doet (Rule of Least Power).

"HTML eerst, dan CSS, dan JS."

JS is zwaarder voor de browser en vaak helemaal niet nodig. Dingen die vroeger niet konden in CSS (zoals input stylen) kunnen dat nu wel. Blijf je kennis updaten. appearance: none maakt form elements volledig stylebar. Vergeet toegankelijkheid niet: focus outlines en prefers-reduced-motion. jQuery lijkt weinig code, maar stuurt alsnog veel mee.

Alternatieven voor JS

  • scroll-behavior: smooth
  • dialog — vervangt JS alerts, styleable met backdrop en @starting-style
  • select en picker
